解決"server certificate verification failed. CAfile..."問題

參考資訊：
1. apt-get-update-failed-because-certificate-verification-failed-because-handshake

問題： 

root@debian:/# apt-get update
    Ign https://old-releases.ubuntu.com quantal InRelease
    Ign https://old-releases.ubuntu.com quantal Release.gpg
    Ign https://old-releases.ubuntu.com quantal Release
    Ign https://old-releases.ubuntu.com quantal/main armel Packages/DiffIndex
    Ign https://old-releases.ubuntu.com quantal/main Translation-en
    Err https://old-releases.ubuntu.com quantal/main armel Packages
      server certificate verification failed. CAfile: /etc/ssl/certs/ca-certificates.crt CRLfile: none
    W: Failed to fetch https://old-releases.ubuntu.com/ubuntu/dists/quantal/main/binary-armel/Packages  server certificate verification failed. CAfile: /etc/ssl/certs/ca-certificates.crt CRLfile: none

    E: Some index files failed to download. They have been ignored, or old ones used instead.

解法： 

# touch /etc/apt/apt.conf.d/99verify-peer.conf && echo >>/etc/apt/apt.conf.d/99verify-peer.conf "Acquire { https::Verify-Peer false }"
