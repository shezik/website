OpenSCAD

開發環境

基本上，畫圖的工具很多，每個人都有各自喜愛的原因，對於司徒來說，利用程式語言來表達圖形還是比較直觀的方式，司徒只要輸入程式，然後編譯就可以馬上看到結果，比起其它需要學習基礎GUI設定的工具，司徒個人還是喜愛程式語言的設計方式，當然，執行的平台還是依賴在Debian系統上，畢竟Debian還是司徒比較熟悉的環境，也是司徒每天使用的系統，而安裝的方式相當簡單，只要使用如下指令即可： 

$ sudo apt-get update
$ sudo apt-get install openscad -y

執行後的UI

如果使用者比較喜愛使用Python程式語言，可以嘗試FreeCAD這套軟體，FreeCAD的特色就是結合UI工具的交互設計方式，而OpenSCAD則是必須完全使用程式語言設計，無法透過UI工具做微調。

為何司徒想要學習OpenSCAD呢？原因就是3D Printer需要使用到，司徒相當喜愛亂改造東西，但是，改造後的東西都是丟掉居多，原因就是手動黏合的東西，真的有點醜，加上司徒裁切的尺寸不夠精準，因此，體積通常比較大，這也造成司徒不願意攜帶使用的原因，而得助於3D印表機的快速發展，價格相對便宜且精準度已經提昇很多，人人都可以是設計師，因此，如何結合自己的設計想法並且利用3D印表機來實現呢？答案就是：繪圖軟體，這也是司徒選擇OpenSCAD的原因，司徒透過靈感撰寫OpenSCAD程式，接著編譯產生STL檔案，最後再藉由3D印表機列印這個STL檔案，完成自己的作品。
