逆向MFC編譯的程式

基本上，逆向MFC編譯的程式會比一般Win32 API程式來得困難，原因在於MFC封裝了Win32 API，而程式主體都是以訊息為溝通媒介，因此處理訊息的Callback副程式比較難找，因此，需要使用OllyDbg和IDA Pro一起定位，這樣就可以找出GUI元件對應的處理副程式位置，司徒使用Dingoo A320的Unbrick解說一下過程：

接著用OllyDbg載入

開啟Executable Modules頁面並且Double Click MFC42.DLL

位於MFC42.DLL領空

Ctrl + N開啟API清單並Double Click OnCmdMsg@CCmdTarget副程式(MFC主要處理訊息入口)

OnCmdMsg進入點

F2設定斷點

接著按F9執行

往下找到AfxFindMessageEntry

Shift + F2設定條件斷點，EBX就是訊息種類0x111 = WM_COMMAND

設定後的顏色

接著將進入點的中斷拿掉，不然會一直中斷

接著按F9並且在下圖按鈕上按一下

斷點位於AfxFindMessageEntry(WM_COMMAND)

按一下F8，接著就可以在EAX上看到該處理副程式的位置

接著開啟IDA Pro並載入Unbrick程式，跳到該位址

Message Structure

原定義 

struct AFX_MSGMAP_ENTRY {
  int nMessage;
  int nCode;
  int NID;
  int nLastID;
  int* NSIG;
  void* PFN;
};

PFN就是指到副程式位置
